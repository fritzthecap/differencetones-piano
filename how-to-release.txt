- Edit pom.xml and remove the '-SNAPSHOT' from <version> XML-element and save file

- Open terminal window and go into the project-directory where Maven pom.xml file is

- Remove old JAR files from compile target directory:
    rm target/differencetones-piano-*.jar
    
- Launch:
    mvn package
    
- Check that all unit-tests were successful and the JAR file in target has no SNAPSHOT postfix

- Copy generated JAR file in target to project directory, leaving out version: 
    cp target/differencetones-piano-*.jar ./differencetones-piano.jar
    
- Check that correct version is in title-bar of application by launching it:
    java -jar differencetones-piano.jar
    
- Commit and push all project changes to repository

- For major releases, create a release on github ("Releases" - "Draft a New Release"),
   attach the differencetones-piano.jar file to the release, and
   edit README.md and adjust the release version number in hyperlink.
   Then fetch / pull the README.md change to local disk!
